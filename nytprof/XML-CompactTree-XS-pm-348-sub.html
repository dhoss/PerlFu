65=head1 DESCRIPTION
66
67This module provides functions that use XML::LibXML::Reader to parse
68an XML document into a parse tree formed of nested arrays (and hashes). It
69aims to be very fast in doing that and to presreve all relevant
70information from the XML (including namespaces, document order, mixed
71content, etc.). It sacrifices user friendliness for speed.
72
73=head1 PURPOSE
74
75I wrote this module because I noticed that repeated calls to methods
76implemented in C (XS) were very expensive in Perl.
77
78Therefore traversing a large DOM tree using XML::LibXML or iterating
79over an XML stream using XML::LibXML::Reader was much slower than
80traversing similarly large and structured native Perl data
81structures.
82
83This module allows the user to do just one call XS within which a
84document parse tree built using native Perl data structures (arrays
85and optionally hashes). It does not provide full DOM navigation but
86attempts to provide maximum amount of information. Its memory
87footprint should be somewhat smaller than that of a corresponding
88XML::LibXML DOM tree.
89

169=head1 OUTPUT FORMAT
170
171The result of parsing a subtree is a Perl array reference C<$node>
172contains a node type followed by node data whose interpretation on
173further positions in $node depends on the node type, as described
174below:
175
176=head2 Any Node
177
178=over 5
179
180=item *
181
182$node->[0] is an integer representing the node type. Use
183XML::LibXML::Reader node-tye constants, e.g. XML_READER_TYPE_ELEMENT
184for an element node, XML_READER_TYPE_TEXT for text node, etc.
185
186=back
187
188=head2 Document or Document Fragment Nodes
189
190=over 5
191
192=item *
193
194$node->[1] contains the document encoding
195
196=item *
197
198$node->[2] is an array reference containing similar represention of
199all the child nodes of the document (fragment).
200
201=back
202
203Note: XML::LibXML::Reader does not document node by default, which
204means that calling readSubtreeToPerl on a reader object in its initial
205state only parses the first node in the document (which can be the
206root element, but also a comment or a processing instruction). Use
207XCT_DOCUMENT_ROOT flag to force creating a document node in such case.
208
209=head2 Element nodes
210
211=over 5
212
213=item *
214
215$node->[1] is the local name (UTF-8 encoded character string)
216
217=item *
218
219$node->[2] is the namespace index (see L<NAMESPACES> below)
220
221=item *
222
223$node->[3] is undef if the element has no attributes. Otherwise if
224XCT_ATTRIBUTE_ARRAY flag was used, $node->[3] is an array reference of
225the form C<[ name1, value1, name2, value2, ....]> of attribute names and
226corresponding values. If XCT_ATTRIBUTE_ARRAY flag was not used, then
227$node->[3] is a hash reference mapping attribute names to the
228corresponding attribute values C<{ name1=>value1, name2=>value2...}>
229
230The flag XCT_KEEP_NS_DECLS controls whether namespace declarations
231(xmlns=... or xmlns:prefix=...) are included along with normal
232attributes or not.
233
234Note: there is no support for namespaced attributes yet, but the
235attribute names are stored as QNames, so one can always use
236XCT_KEEP_NS_DECLS to keep track of namespace prefix declarations and
237do the resolving manually. Support for namespaced attributes is
238planned.
239
240=item *
241
242If XTC_LINE_NUMBERS flag was used, $node->[4] contains the line number
243of the element and $node->[5] contains an array reference containing
244similar representions of the child nodes of the current node.
245
246=item *
247
248If XTC_LINE_NUMBERS flag was NOT used, $node->[4] contains an array
249reference of similar representations of the child nodes of the current
250node.
251
252=back
253
254=head2 Text, CDATA, Comment and White-Space Nodes
255
256=over 5
257
258=item *
259
260$node->[1] contains the node value (UTF-8 encoded character string)
261
262=back
263
264=head2 Unparsed Entity, Processing-Instruction, and Notation Nodes
265
266=over 5
267
268=item *
269
270$node->[1] contains the local name (there is no support for
271namespaces on these types of nodes yet)
272
273=item *
274
275$node->[2] contains the node value
276
277=back
278
279=head2 Skipping Less-Significant Nodes
280
281White-space (non-significant or significant), processing-instruction
282and comment nodes can be completely skipped, using the following
283flags:
284
285 XCT_IGNORE_WS
286 XCT_IGNORE_SIGNIFICANT_WS
287 XCT_IGNORE_PROCESSING_INSTRUCTIONS
288 XCT_IGNORE_COMMENTS
289
290=head1 NAMESPACES
291
292Namespaces of element nodes are stored in the element node as an
293integer. 0 always represents nodes without namespace, all other
294namespaces are assigned unique numbers in an increasing order as they
295appear. You can pass an empty hash reference to the parsing functions
296to obtain the mapping.
297
298=head2 Example
299
300 use XML::CompactTree::XS;
301 use XML::LibXML::Reader;
302
303 my $reader = XML::LibXML::Reader->new(location => $ARGV[0]);
304 my %ns;
305 my $data = XML::CompactTree::XS::readSubtreeToPerl( $reader, XCT_DOCUMENT_ROOT, \%ns );
306 $ns_map[$ns{$_}]=$_ for keys %ns;
307 my @nodes = ($data);
308 while (@nodes) {
309 my $node = shift @nodes;
310 my $type = $node->[0];
311 if ($type == XML_READER_TYPE_ELEMENT) {
312 print "element $node->[1] is from ns $node->[2] '$ns_map[$node->[2]]'\n";
313 push @nodes, @{$node->[4]}; # queue children
314 } elsif ($type == XML_READER_TYPE_DOCUMENT) {
315 push @nodes, @{$node->[2]}; # queue children
316 }
317 }

319=head1 PLANNED FEATURES
320
321Planned flags:
322
323 XCT_USE_QNAMES - use QNames instead of local names for all nodes
324 XCT_TEXT_AS_STRING - put text nodes into the tree as plain scalars
325 XCT_PRESERVE_PARENT - add a slot with a weak reference to the parent node
326 XCT_MERGE_TEXT_NODES - merge adjacent text/cdata nodes together
