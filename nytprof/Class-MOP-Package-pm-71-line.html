267=head1 NAME
268
269Class::MOP::Package - Package Meta Object
270
271=head1 VERSION
272
273version 2.0401
274
275=head1 DESCRIPTION
276
277The Package Protocol provides an abstraction of a Perl 5 package. A
278package is basically namespace, and this module provides methods for
279looking at and changing that namespace's symbol table.
280
281=head1 METHODS
282
283=over 4
284
285=item B<< Class::MOP::Package->initialize($package_name, %options) >>
286
287This method creates a new C<Class::MOP::Package> instance which
288represents specified package. If an existing metaclass object exists
289for the package, that will be returned instead. No options are valid at the
290package level.
291
292=item B<< Class::MOP::Package->reinitialize($package, %options) >>
293
294This method forcibly removes any existing metaclass for the package
295before calling C<initialize>. In contrast to C<initialize>, you may
296also pass an existing C<Class::MOP::Package> instance instead of just
297a package name as C<$package>.
298
299Do not call this unless you know what you are doing.
300
301=item B<< Class::MOP::Package->create($package, %options) >>
302
303Creates a new C<Class::MOP::Package> instance which represents the specified
304package, and also does some initialization of that package. Currently, this
305just does the same thing as C<initialize>, but is overridden in subclasses,
306such as C<Class::MOP::Class>.
307
308=item B<< Class::MOP::Package->create_anon(%options) >>
309
310Creates a new anonymous package. Valid keys for C<%options> are:
311
312=over 4
313
314=item C<weaken>
315
316If this is true (the default), the instance stored in C<Class::MOP>'s metaclass
317cache will be weakened, so that the anonymous package will be garbage collected
318when the returned instance goes out of scope.
319
320=back
321
322=item B<< $metapackage->is_anon >>
323
324Returns true if the package is an anonymous package.
325
326=item B<< $metapackage->name >>
327
328This is returns the package's name, as passed to the constructor.
329
330=item B<< $metapackage->namespace >>
331
332This returns a hash reference to the package's symbol table. The keys
333are symbol names and the values are typeglob references.
334
335=item B<< $metapackage->add_package_symbol($variable_name, $initial_value) >>
336
337This method accepts a variable name and an optional initial value. The
338C<$variable_name> must contain a leading sigil.
339
340This method creates the variable in the package's symbol table, and
341sets it to the initial value if one was provided.
342
343=item B<< $metapackage->get_package_symbol($variable_name) >>
344
345Given a variable name, this method returns the variable as a reference
346or undef if it does not exist. The C<$variable_name> must contain a
347leading sigil.
348
349=item B<< $metapackage->get_or_add_package_symbol($variable_name) >>
350
351Given a variable name, this method returns the variable as a reference.
352If it does not exist, a default value will be generated if possible. The
353C<$variable_name> must contain a leading sigil.
354
355=item B<< $metapackage->has_package_symbol($variable_name) >>
356
357Returns true if there is a package variable defined for
358C<$variable_name>. The C<$variable_name> must contain a leading sigil.
359
360=item B<< $metapackage->remove_package_symbol($variable_name) >>
361
362This will remove the package variable specified C<$variable_name>. The
363C<$variable_name> must contain a leading sigil.
364
365=item B<< $metapackage->remove_package_glob($glob_name) >>
366
367Given the name of a glob, this will remove that glob from the
368package's symbol table. Glob names do not include a sigil. Removing
369the glob removes all variables and subroutines with the specified
370name.
371
372=item B<< $metapackage->list_all_package_symbols($type_filter) >>
373
374This will list all the glob names associated with the current
375package. These names do not have leading sigils.
376
377You can provide an optional type filter, which should be one of
378'SCALAR', 'ARRAY', 'HASH', or 'CODE'.
379
380=item B<< $metapackage->get_all_package_symbols($type_filter) >>
381
382This works much like C<list_all_package_symbols>, but it returns a
383hash reference. The keys are glob names and the values are references
384to the value for that name.
385
386=item B<< Class::MOP::Package->meta >>
387
388This will return a L<Class::MOP::Class> instance for this class.
389
390=back
