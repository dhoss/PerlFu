172=head1 NAME
173
174Net::Kestrel - Kestrel Client for Perl
175
176=head1 VERSION
177
178version 0.08
179
180=head1 SYNOPSIS
181
182 use Net::Kestrel;
183
184 my $queuename = 'myqueue';
185
186 my $kes = Net::Kestrel->new; # defaults to host => 127.0.0.1, port => 2222
187 $kes->put($queuename, 'foobar');
188 # ... later
189
190 # take a peek, doesn't remove the item
191 my $item = $kes->peek($queuename);
192
193 # get the item out, beginning a transaction
194 my $real_item = $kes->get($queuename);
195 # ... do something with it
196
197 # then confirm we finished it so kestrel can discard it
198 $kes->confirm($queuename, 1); # since we got one item
199
200=head1 DESCRIPTION
201
202Net::Kestrel is a B<text protocol> client for L<https://github.com/robey/kestrel>.
203
204=head1 ATTRIBUTES
205
206=head2 debug
207
208=head2 host
209
210The ip address of the Kestrel host you want to connect to.
211
212=head2 port
213
214The port to connect to. Defaults to 2222.
215
216=head2 timeout
217
218The timeout value for operations. Defaults to 3 seconds.
219
220=head1 METHODS
221
222=head2 confirm ($queuename, $count)
223
224Confirms $count items from the queue.
225
226=head2 delete ($queuename)
227
228Delete the specified queue.
229
230=head2 flush ($queuename)
231
232Flush (empty) the specified queue.
233
234=head2 get ($queuename, $timeout_in_millis)
235
236Gets an item from the queue. Note that this implicitly begins a transaction
237and the item must be C<confirm>ed or kestrel will give the item to another
238client when you disconnect. Optionally you may provide a timeout (in
239milliseconds). Net::Kestrel will block for that long waiting for a value in
240the queue.
241
242=head2 peek ($queuename, $timeout_in_millis)
243
244Peeks into the specified queue and "peeks" at the next item. Optionally you
245may provide a timeout (in milliseconds). Net::Kestrel will block for that
246long waiting for a value in the queue.
247
248=head2 put ($queuename, $string)
249
250Puts the provided payload into the specified queue.
251
252=head2 stats
253
254Returns stats from the kestrel instance
255
256=head1 NOTES
257
258=head2 Incomplete
259
260B<This module is brand new and is likely missing features.>
261
262=head2 Protocol
263
264Net::Kestrel speaks Kestrel's text protocol only at present.
265
266=head2 Error Handling
267
268Kestrel returns errors in the form of:
269
270 -Error string
271
272When any command returns a string like this, Net::Kestrel will die with that
273message. Therefore you should C<eval> any methods you care to deal with
274errors for.
